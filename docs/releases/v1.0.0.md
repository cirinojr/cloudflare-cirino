# Cloudflare Cirino v1.0.0 (Draft Release Notes)

Initial public release of Cloudflare Cirino.

## Summary

Cloudflare Cirino adds predictable Cloudflare cache invalidation to WordPress content workflows without requiring external services or a build pipeline.

## What’s included

- Automatic purge on relevant public content lifecycle events.
- Purge modes:
  - Targeted URLs (permalink + related URLs)
  - Purge Everything (full zone)
- Authentication modes:
  - API Token (recommended)
  - Legacy Email + Global API Key
- Native wp-admin settings screen under Tools.
- Manual actions for connection testing and immediate purge.
- Runtime metadata and a short recent activity log.

## Technical notes

- Built on WordPress HTTP API and standard options/settings APIs.
- Admin actions are protected via capability checks and nonces.
- Autosaves and revisions are ignored to avoid unnecessary purge traffic.

## Upgrade / install notes

- Requires WordPress 6.1+
- Requires PHP 7.4+
- Configure Zone ID and credentials after activation.

## Known limitations in v1.0.0

- Purges are executed inline (no async queue).
- No multisite network admin screen.
- No WP-CLI integration yet.
